# 光影手记 / SNAPSHOT — 项目设计上下文

## Design Context

### Users

- **访客（普通浏览者）**：摄影爱好者、文青、朋友，访问网站的目的是欣赏照片、发现照片背后的故事
- **管理员（博主自己）**：热爱摄影，希望打造一个独特的个人摄影展示空间，便于管理作品

**使用场景**：用户在碎片化时间（午休、通勤、睡前）打开网站，希望能获得放松、治愈的视觉体验

### Brand Personality

- **诗意**：每一张照片都是"时间的容器"，承载着拍摄瞬间的情绪和故事
- **温暖**：像日记本一样亲切、温暖，而非冰冷的相册展示
- **惊喜**：随机光影交互带来发现的乐趣，点击查看心情文字带来故事性惊喜

**情感目标**：让用户感受到"原来这张照片背后有这样的故事"，带来温暖和治愈感

### Aesthetic Direction

**整体视觉**：
- 暖色调/胶片感，类似复古相纸的质感
- 浅色模式为主，仿日记本的纸张质感
- 深色模式可选，模拟暗房氛围

**配色方案（推荐）**：
- 主色调：暖米色、浅棕色（仿旧纸张）
- 强调色：琥珀色、焦糖色（温暖点缀）
- 背景：米白色、浅灰（柔和不刺眼）

**字体方案（推荐）**：
- 正文/衬线体：思源宋体（Noto Serif）或 Source Han Serif
- 心情文字：手写体风格，如站酷快乐体、或衬线体斜体模拟手写感
- 标题：无衬线 + 手写混搭

**反参考**：避免过度现代化（纯黑纯白）、避免复杂的渐变和3D效果

### Design Principles

1. **故事性优先**：每一个交互都在引导用户发现照片背后的故事
2. **惊喜感**：随机光影交互 + 心情文字弹出，给用户"不知道会发现什么"的期待
3. **氛围感**：整体视觉要像翻开一本旧日记本，温暖、亲切、有质感
4. **克制简洁**：不过度装饰，让照片本身成为主角
5. **流畅动效**：过渡动画柔和自然，增强"翻阅"感

## 技术上下文

### 技术栈

- 前端：Vue3 + TypeScript + Vite
- 后端：Node.js + Express
- 数据库：MySQL 8.x

### 核心功能模块

| 模块 | 功能 |
|------|------|
| 摄影作品展示 | 照片网格/瀑布流、划过随机高亮、点击查看心情文字、懒加载 |
| 照片管理 | 管理员登录、照片上传/编辑/删除、心情文字管理 |
| 浏览模式 | 时间轴视图、随机日记、收藏精选 |
